# GitHub Pages Website für selbstständige IT-Beratung

## Dateien
- `index.html` – Landing Page
- `projects.html` – Projektportfolio
- `blog.html` – Blogseite
- `about.html` – Über-mich-Seite
- `styles.css` – zentrales Styling und Animationen
- `script.js` – Navigation, Scroll-Reveal, Counter, Filter, Blogsuche, Partikelanimation

## Veröffentlichung auf GitHub Pages
1. Neues GitHub-Repository anlegen.
2. Alle Dateien hochladen.
3. In den Repository-Einstellungen `Pages` öffnen.
4. Als Source den Branch `main` und den Root-Ordner auswählen.
5. Speichern – danach wird die Seite auf `github.io` veröffentlicht.

## Dinge, die du direkt anpassen solltest
- Ersetze `Dein Name` in allen HTML-Dateien.
- Passe die Links für LinkedIn, Instagram und GitHub an.
- Ergänze reale Projektdetails, Repositories und Screenshots.
- Erweitere Blogartikel oder füge neue Einträge hinzu.
- Schärfe die Formulierungen in `about.html` auf dein echtes Profil.

## Blog erweitern
Du kannst neue Artikel sehr einfach hinzufügen, indem du in `blog.html` weitere `article.blog-card`-Blöcke kopierst und anpasst.

## Hinweis
Die Seite ist bewusst ohne Framework gebaut, damit sie stabil und unkompliziert auf GitHub Pages läuft.
